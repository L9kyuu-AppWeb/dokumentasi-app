Bertindaklah sebagai UI/UX Designer, Technical Writer, Product Marketing, dan Software Architect profesional.

Saya telah menyelesaikan sebuah aplikasi dan ingin membuat website dokumentasi yang bukan hanya sebagai panduan penggunaan, tetapi juga sebagai media promosi agar pengunjung tertarik menggunakan aplikasi saya.

Target dokumentasi:
- Profesional
- Modern
- Clean
- Mudah dipahami
- Meyakinkan calon pengguna
- Cocok dijadikan landing page produk

Gunakan gaya bahasa formal namun mudah dipahami.

Buat dokumentasi dengan struktur berikut.

# 1. Hero Section
Berisi:
- Nama aplikasi
- Tagline
- Deskripsi singkat
- Tombol CTA:
  - Demo
  - Dokumentasi
  - Hubungi
  - GitHub (opsional)
- Screenshot Hero

---

# 2. Tentang Aplikasi

Jelaskan:
- Latar belakang
- Permasalahan yang diselesaikan
- Tujuan aplikasi
- Target pengguna
- Manfaat utama

---

# 3. Keunggulan Aplikasi

Minimal 6 poin.

Setiap poin harus memiliki:
- Icon
- Judul
- Penjelasan manfaat

Contoh:
✔ Cepat
✔ Aman
✔ Responsive
✔ Multi User
✔ Mudah digunakan
✔ Modern UI

---

# 4. Fitur Utama

Buat dalam bentuk card.

Setiap fitur terdiri dari:
- Icon
- Nama fitur
- Deskripsi manfaat
- Screenshot (placeholder)

Jangan hanya menjelaskan fungsi, tetapi juga manfaat bagi pengguna.

---

# 5. Screenshot Aplikasi

Buat galeri screenshot untuk:

- Login
- Dashboard
- Master Data
- Transaksi
- Laporan
- Pengaturan

Setiap screenshot memiliki:
- Judul
- Penjelasan singkat

---

# 6. Video Demo

Sediakan section video demo yang berisi:
- Thumbnail
- Tombol Play
- Deskripsi video

---

# Output yang diharapkan

- Susunan section yang rapi
- Heading yang menarik
- Deskripsi profesional
- Copywriting yang mampu menarik pengguna
- Gunakan gaya seperti website dokumentasi produk SaaS modern (misalnya Laravel, Supabase, Vercel, atau Figma)
- Sertakan placeholder untuk gambar, video, diagram, dan ikon agar mudah diganti dengan aset asli.
